# Hospitality OS Video Production SOP

## Status flow
Discovery Needed → Brief Ready → Script Draft → Internal Review → Client Approval → In Production → Quality Review → Scheduled → Published → Measured → Archived

## Procedure

1. **Discovery** — collect client, objective, audience, offer, proof, language, channel, CTA, destination, deadline, and approval owner.
2. **Fact gate** — stop when prices, dates, capacity, availability, or claims are unverified.
3. **Script package** — run the master prompt and save the version.
4. **Internal review** — check hook, pacing, factual accuracy, brand voice, and CTA alignment.
5. **Client approval** — record script and language approval before production.
6. **Production** — generate the draft with the authorized avatar, voice, format, and approved assets.
7. **Quality review** — watch with sound and muted; inspect lip sync, pronunciation, subtitles, safe margins, visuals, music, and CTA.
8. **Publication** — test the destination on mobile, publish to the approved channel, and record the live URL.
9. **Measurement** — capture results after 24 hours, 7 days, and 30 days where relevant.
10. **Learning** — document the winning hook, language, audience, CTA, and next experiment.

## Security
- Never commit API keys or private avatar identifiers.
- Store identity footage and consent records outside public repositories.
- Never create deceptive testimonials or unauthorized impersonations.
- Human or client approval is mandatory before public release.

## Failure routing
- Missing offer details → Sales Operator or client owner
- Missing event details → Event Booking Agent
- Weak distribution plan → Social Media Agent
- Language uncertainty → approved human reviewer
- Broken destination or lead flow → Delivery Manager
- Negative guest situation → Customer Recovery Agent
