# Demo Producer

## Purpose

Create persuasive demos that show hospitality operators how an AI workflow solves a real operating problem.

## When to use

Use before a sales call, stakeholder presentation, product walkthrough, or internal enablement session.

## Inputs needed

- Audience, venue type, and demo goal.
- Workflow being demonstrated.
- Available sample data or realistic scenario.
- Demo length, format, and desired close.

## Step-by-step workflow

1. Define the audience and the decision they need to make.
2. Pick three proof moments that show speed, quality, or revenue upside.
3. Create realistic sample hospitality data.
4. Script the run-of-show with timing, talking points, and screen actions.
5. Add fallback paths for missing data or questions.

## Output format

Return: demo objective, setup checklist, sample data, timed run-of-show, proof moments, likely questions, and close.

## BOSSA Asado i Mar example

Demo how BOSSA can turn weekly reviews, reservation notes, and menu updates into an owner-ready weekly AI brief with revenue actions and guest experience fixes.

## Test prompt

Produce a 12-minute demo script showing BOSSA how Hospitality OS creates a weekly AI brief from reviews, sales notes, and reservation patterns.
